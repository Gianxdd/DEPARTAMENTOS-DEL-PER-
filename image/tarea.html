Respondiendo preguntas :

1.¿QUÉ ES UN PROTOCOLOS HTTP?

Es el nombre de un protocolo que nos permite solicitar datos y recursos, como documentos HTML. Es la base de todo intercambio de datos en la web y es un protocolo de la arquitectura cliente-servidor, lo que significa que las solicitudes de datos son iniciadas por el elemento receptor, generalmente un navegador web. Así, una página web completa es el resultado de una combinación de varios subdocumentos recibidos, tales como: documentos que especifican el estilo de diseño del sitio, texto, imágenes, videos, scripts, etc. . ..

2.¿QUÉ ES UN QUERY STRING?

En el protocolo HTTP podemos encontrar diferentes métodos (verbos) con los que podemos realizar diferentes tipos de solicitudes. El método más utilizado es definitivamente el método GET, un método que te permite obtener un recurso del servidor, ya sea una página web, un archivo txt, un gif, etc.

3.¿QUÉ ES UN VERBOS HTTP?

Los verbos HTTP relacionados con el sistema REST son GET, POST, PUT, PATCH y DELETE. GET es lo que usamos para consultar recursos. Una de las principales características de una solicitud GET es que no debe causar efectos secundarios en el servidor, no debe crear nuevos registros ni modificar registros existentes.

4.¿QUÉ ES UN DNS?

El Sistema de nombres de dominio (DNS) es la guía telefónica de Internet. Las personas acceden a la información en línea a través de dominios. Los navegadores web interactúan a través de direcciones de Protocolo de Internet (IP). DNS traduce los nombres de dominio en direcciones IP para que los navegadores puedan descargar recursos de Internet.

5.¿QUE ES UN CDN?

Es un grupo de servidores repartidos en varios sitios web. Estos servidores almacenan copias de datos duplicados para que los servidores puedan responder a las solicitudes de datos en función de los servidores más cercanos a los respectivos usuarios finales..
